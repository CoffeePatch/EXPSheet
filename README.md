# EXPSheet

EXPSheet is a comprehensive, multi-platform personal finance and expense tracking system centered around a Google Sheets ledger (`List`). The system is split into three workflow-specific areas under the `scripts` folder.

**AppSheet is the primary, recommended intake system.** The AppSheet backend utilizes the powerful "PORTFOLIO NEXUS Sweeper Engine" to automatically normalize and process all budget data.

| Path | Purpose | Main file |
|---|---|---|
| [scripts/appsheet/](scripts/appsheet/) | **(Primary Workflow)** AppSheet-captured transaction sweeper. Handles split bills, internal transfers, and standard expenses seamlessly. | [IndTran.gs](scripts/appsheet/IndTran.gs) |
| [scripts/form-submission/](scripts/form-submission/) | **(Alternative/Legacy Workflow)** Google Forms intake workflow and response processing. | `FORM.gs` |
| [scripts/reconciliation/](scripts/reconciliation/) | Manual bank reconciliation tool to verify your ledger against a pasted bank export. | [Reconciliation.gs](scripts/reconciliation/Reconciliation.gs) |

## Shared Ledger Overview

All workflows revolve around the same spreadsheet ledger (`List`). The current optimized schema utilizes the following key columns:

- `ID`: Unique transaction ID generated by AppSheet.
- `Date`: Date of the transaction.
- `Account`: Source account.
- `Amount`: Numerical amount.
- `Debt Entity` (or `Person`): Used to assign the transaction to a person or entity. Trigger for Split Bills.
- `Category`: Categorization. Triggers Transfer logic when set to specific terms.
- `Notes`: Additional text.
- `Status`: `Settled`, `Pending`, or `Completed`.
- `Settlement Date`: Tracked when pending debts are settled.
